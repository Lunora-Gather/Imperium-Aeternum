# Imperium Aeternum — 用户体验超仔细审查报告 v1

> 审查日期：2026-06-24
> 审查范围：全部 12 个界面 + App 外壳 + store + UI 组件 + LogToast
> 审查视角：**玩家**。不以"代码对不对"为标准，以"玩家会不会困惑/犯错/流失"为标准。
> 严重度：☠ 崩溃 / ⛔ 阻断流程 / ⚠ 体验损伤 / 💡 可优化

---

## 一、已修复的崩溃 bug（本轮）

### ☠ WorldMap.tsx line 63 — `player` 未定义，切换到「舆图」页即崩溃

```ts
// 修复前：const { state } = useGameStore(); —— player 从未解构
const playerArmyHere = isPlayer ? player.army.find(...) : null;  // ← ReferenceError
```

**玩家影响**：点击导航「舆图」或按 `m` 键，整页白屏崩溃。这是 MVP 最严重的体验断点——玩家第一次想看地图就直接死。

**修复**：已加 `const player = state.nations[pid];`。`npx tsc --noEmit` 通过。

**根因反思**：E22 加军队位置标记时引入了 `player`，但 WorldMap 历史上只解构 `state`。说明改动后**没有逐 tab 手动跑一遍**。建议后续加一个冒烟测试：每个 tab 至少渲染一次不抛异常。

## 二、阻断流程级问题（⛔）

### ⛔ 1. 剧本选择页点击剧本时同时触发 `startScenario` 和 `setSelected`，状态错乱

`ScenarioSelect.tsx` line 87：

```tsx
onClick={() => { setSelected(s.id); setPickedNation(null); startScenario(s.id); if (s.id === 'classic') return; }}
```

**玩家影响**：
- 点 `classic` → `startScenario('classic')` 立即开游戏，但同时 `setSelected('classic')` 也设了，组件会进入"选国界面"分支（classic 的 `needsNationPick=false` 所以不会，但逻辑混乱）。
- 点非 classic 剧本 → 同时调 `startScenario`（store 里可能切 scene 到 playing）又 `setSelected`。如果 `startScenario` 不切 scene 只初始化数据，那 OK；如果切了，玩家看到的是 playing 但组件还停在 menu 渲染选国页 → 白屏或双重渲染。
- 这是典型的"一个 onClick 干两件互相矛盾的事"。

**建议**：明确两步分离——先 `setSelected` 进入选国页（不调 startScenario），玩家确认国后再 `startWithNation`。classic 才一步直开。需读 store 的 `startScenario` 确认它是否切 scene 才能定夺，但当前调用顺序本身就是隐患。

### ⛔ 2. 回合结算后强制跳「年报」页，玩家无法回到刚才操作的页面

`App.tsx` line 61-66：

```tsx
useEffect(() => {
  if (justProcessedTurn) { setTab('report'); clearTurnFlag(); }
}, [justProcessedTurn, clearTurnFlag]);
```

**玩家影响**：玩家在「省份」页调税率、建设、征兵后按空格下一回合——结算完被强制弹到「年报」。看完年报想回省份页继续建设，得手动点回去。**每回合都被打断一次操作流**，回合越多越烦。

**建议**：保留自动跳年报（首回合教学好），但加一个"记住结算前所在 tab"的逻辑：结算后跳年报，年报页顶部加一个「← 返回省份」按钮直接回结算前的 tab。或更简单：只在首回合强制跳，之后留在原页，年报用顶部红点提示「今年有新报告」。

### ⛔ 3. 事件弹窗无法用键盘选择，只能鼠标点

`EventModal.tsx` 选项按钮无键盘快捷键（1/2/3 选选项、Esc 关闭也不行）。

**玩家影响**：事件是游戏灵魂，150 个事件每回合可能弹多个。每个都要鼠标移动点击，回合多了手累。App 已有数字键切 tab 的快捷键体系，事件弹窗却没接入。

**建议**：选项按钮加 `1`/`2`/`3` 快捷键（在按钮文字前显示 `[1]`），Esc 暂不可关（事件必须选）。

## 三、体验损伤级问题（⚠）

### ⚠ 4. Dashboard 趋势图战时区块缩进异常，看起来像 bug

`Dashboard.tsx` line 283-290：战时趋势 `{warCount > 0 && (...)}` 嵌在 `hist.length >= 2` 的 fragment 内，但缩进比上面四宫格多一层，视觉上像"贴在了四宫格卡片右下角"而不是独立一块。

**玩家影响**：不明显功能 bug，但视觉层次乱——玩家会以为战时趋势是四宫格的子元素。

**建议**：把战时趋势块移出 `hist.length >= 2` 的同一 grid 容器，独立成一块。

### ⚠ 5. 操作即时反馈 LogToast 只显示最新 1 条且 2.5 秒消失，多个操作会丢消息

`LogToast.tsx`：只看 `log[log.length-1]`，新消息覆盖旧消息，2.5 秒淡出。

**玩家影响**：玩家一回合可能连续点"建设农田→征兵→安抚派系"3 个操作，每操作都 log 一条。第 2 条还没看完就被第 3 条覆盖。玩家不知道前两个操作到底成没成。

**建议**：Toast 改为队列——最多堆 3 条，每条 2.5 秒后逐条消失。或至少在"操作失败"（如行动点不足）时用红色 Toast 且停留更久（4 秒），区别于成功反馈。

### ⚠ 6. 「行动点不足」是最高频的错误，但提示太弱

所有操作失败都走 `get().logMsg('行动点不足')`——一条灰色 Toast 2.5 秒消失。

**玩家影响**：行动点（adminPt）是每回合硬上限，玩家前期会反复撞墙。每次撞墙只看到右下角一闪而过的灰字，很容易没注意，然后以为按钮坏了/没反应。

**建议**：
- 行动点不足时按钮本身变灰 disabled + tooltip 显示「需 2 行动点，当前 1」。
- Toast 用红色 `warn` 样式，且不自动消失直到玩家下一次操作。
- Dashboard 顶部 adminPt 卡片在不足时高亮闪烁。

### ⚠ 7. 省份页「建设」按钮列表一次性显示全部 24 个建筑，无筛选，新手懵

`ProvinceScreen.tsx` line 119-134：`Object.keys(BUILDINGS).map` 把 24 个建筑全平铺，每个只有名字+成本+收益，无前置科技说明。

**玩家影响**：第一回合打开省份页，看到 24 个建设按钮一排排，不知道该建什么、能建什么、为什么有的建了没用。高端建筑（great_library 需 Lv6 科技）和基础农田混在一起，玩家建了高端建筑发现没收益会困惑。

**建议**：
- 按科技解锁分两栏：当前可建（高亮）/ 待解锁（灰且标注「需农业 Lv3」）。
- 或加一个筛选条：全部 / 农业 / 军事 / 行政 / 文化。

### ⚠ 8. 税率调整没有「预估民心影响」，玩家不敢动

`EconomyScreen.tsx` 税率滑块只显示当前税率和建议档文字，但调高税率会降多少民心、调低会少多少金——玩家看不到。

**玩家影响**：税率是经济系统核心杆，但玩家调高怕崩民心、调低怕赤字，最后一直放默认 15% 不敢动。这正是设计要避免的"杆没人敢拉"。

**建议**：滑块旁加一个实时预估：「当前 15% → 若调至 25%：年税 +X 金，民心 -Y」。民心影响用公式算出来给玩家看，消除盲调。

### ⚠ 9. 存档页只支持单存档槽，无存档时间戳，玩家不敢覆盖

`SaveLoadScreen.tsx`：只有一个 localStorage 键，存档即覆盖，无时间戳显示。

**玩家影响**：玩家打了 50 回合想存个档，但怕覆盖掉之前的好档，于是不敢存。或者存了之后想回档到 30 回合，做不到——已经被 50 回合覆盖。

**建议**：至少显示「上次存档时间 + 第几年」，让玩家知道覆盖的是哪个档。中期扩展多槽存档（3 个槽）。

### ⚠ 10. 外交关系图节点 24 个挤在一个 400px 圆里，文字看不清

`DiplomacyScreen.tsx` line 28：`.slice(0, 24)`，400px SVG，24 个节点环形排布。

**玩家影响**：205 国剧本里玩家有 24 个外交对象很正常，但挤在 400px 圆周上每个节点间距不到 50px，国家名缩写 2 字都看不清，更别说点击。

**建议**：默认只显示 top 8（按 |关系值|），加一个"展开全部"按钮。或做成可滚动的列表 + 图混排。

## 四、可优化级问题（💡）

### 💡 11. 首回合新手引导卡只在 `state.turn === 0` 显示，读档后不再显示

老玩家读档后想再看一眼引导找不到。建议加一个「？」帮助按钮常驻 header，点开重新显示引导卡。

### 💡 12. ResourceStrip header 资源速览没有「+净收入/年」提示

玩家看 header 国库 1000，不知道下回合会变多少。建议国库数字下加小字 `+42/年` 或 `-15/年`，用上次年报的 net 值。

### 💡 13. 空格键下一回合，但事件未处理时空格也能按 → 回合推进卡死

`App.tsx` line 91：`if (e.code === 'Space') { ... if (!state.victory.type) nextTurn(); }`——没检查是否有 pendingEvents。

**玩家影响**：事件弹窗挡在中间，玩家按空格想关弹窗，结果触发了下一回合，事件没选就被结算了（或事件还挂在 pendingEvents 里，下次回合又弹）。

**建议**：`pendingEvents.some(p => p.nationId === pid)` 时空格禁用 nextTurn，改成 Space 关弹窗/选第一个选项。

### 💡 14. 年报页看完没有「继续」按钮，玩家不知道看完该干嘛

`TurnReportScreen` 只展示数据，无操作引导。新手看完年报愣住：然后呢？

**建议**：年报底部加一个大按钮「→ 进入下一回合」或「← 返回总览」。

### 💡 15. 地图页省份只显示为圆点，玩家无法从地图快速进省份页

`WorldMap.tsx` 省份是 `<circle>`，hover 显示信息，但点击无反应（没 onClick 进省份页）。

**玩家影响**：玩家在地图上看到一个骚动省份想点进去处理，结果只能记名字再去省份页找。

**建议**：circle 加 onClick → 设省份页选中省 + 跳 province tab。

### 💡 16. 骚动省份徽章区「→ 省份页处理」是纯文字提示，不可点

`Dashboard.tsx` line 255：`<span>→ 省份页处理（部署驻军/安抚）</span>` 纯文字。

**建议**：做成可点链接，点了直接跳省份页且选中该骚动省。

### 💡 17. 战争进展和厌战趋势用「当前厌战近似历史」——趋势线是平的

`Dashboard.tsx` line 119：`const exhaustTrend = hist.map(() => player.warExhaustion)`——每回合都填当前值，sparkline 是一条横线，毫无信息量。

**建议**：TurnReport 里加 `warExhaustionSnapshot` 字段，buildReport 时存当时的厌战值，趋势线才有意义。本轮 state summary 已提到"历史无快照"——值得补。

### 💡 18. 科技页「点下方即时研发完成」vs「AI 渐进研发」——玩家不知道这差异的意义

`TechnologyScreen.tsx` line 44：玩家点科技即完成扣点，AI 是渐进。这设计意图不明（是简化玩家操作？还是给玩家优势？）。

**建议**：加一句说明「玩家可即时决策，AI 按回合推进——你比 AI 更灵活」。

### 💡 19. 政策/法律推行失败只 log「推行失败」，不说原因

`PoliticsScreen.tsx` line 24：`if (!ok) logMsg('推行失败')`。为什么失败？行动点不足？前置条件没满足？已推行？

**建议**：store 的 enactPolicy/enactLaw 返回 `{ok, reason}`，失败时显示具体原因。

### 💡 20. 主题切换（昼/夜）只在 menu 外的 App 顶栏，menu 页切不了

ScenarioSelect 是 menu 场景独立渲染，没有 theme 切换按钮。玩家开场想先调主题找不到。

**建议**：menu 页右下角也放一个主题切换。

## 七、总体体验评价

**优点**（保护不丢）：
- 青铜铭文设计语言统一、有质感，夜晚主题氛围足。
- 警报/危机分级（failWarns 倾覆在即 + crisis 红边脉冲）反馈到位，是同类策略游戏少有的紧迫感设计。
- 事件弹窗效果 Tag 预览 + 事件链标记，消除了"盲选"焦虑。
- 和约条件预览（PeaceTermsBtn）同理，减少战争盲点。
- 快捷键体系（数字键切 tab、空格下一回合）骨架在。

**短板**（重点补）：
- **反馈鲁棒性**：操作失败提示太弱、Toast 会丢消息、政策失败无原因——玩家"做了没反应"是最伤的。
- **操作流连贯性**：结算后强制跳页、事件无键盘、地图点不进省份——每回合被打断多次。
- **新手可学性**：24 建筑平铺、税率无预估、科技即时/渐进不解释——杆再多玩家不敢拉就等于没有。
- **信息可读性**：外交图 24 节点挤、厌战趋势平线、存档无时间戳——有数据但玩家读不出来。

**一句话**：系统深度足够，演出氛围在线，但**反馈层和操作流**还差一层打磨——玩家会因"不知道操作成没成"和"每回合被打断"而流失，而非因系统不够深。优先补 P0/P1 的 8 项，体验完整度会上一个台阶。
