# 账号验证与社交体验

> 本文是账号注册、登录反馈、好友入口和版图聊天体验的当前事实源。共享世界结算规则仍以 `SHARED-WORLD.md` 为准。

## 1. 可信账号流程

- 游客可以完整使用单机模式，不强迫登录。
- 新账号必须先填写昵称、邮箱和密码，再由 Appwrite Email OTP 发送 6 位验证码。
- 只有验证码建立已验证会话后，客户端才设置昵称与密码并完成注册。
- 未验证邮箱不能使用密码会话进入应用；可以通过邮箱验证码完成验证并登录。
- 忘记密码时可使用 6 位邮箱验证码建立临时已验证会话；账号网关只接受 10 分钟内新建、包含邮箱因子的 token 会话，并只更新同一用户的既有密码账号。成功后直接安全登录。
- 验证码由 Appwrite 生成和校验，前端不保存验证码、不建立自有验证码表。
- 邮箱只用于认证，不写入公开玩家资料；好友使用独立好友码。

### 状态机

`填写资料 → 发送验证码 → 验证邮箱 → 设置昵称/密码 → 已认证`

切换登录/注册模式或修改资料会清除本地待验证状态。验证码 15 分钟有效，界面限制 60 秒后重发，以降低误触和频繁请求。

## 2. 交互与视觉规则

- 所有主操作必须提供按下位移、键盘焦点、禁用态和异步加载反馈。
- 浏览器密码管理器自动填充必须继承主题背景、文字与焦点环，不得回退为浏览器默认白色或浅蓝色块。
- 表单在输入时显示就地校验；服务端结果使用 `role=status` 通知。
- 注册页明确显示两步进度与“邮箱验证是必需步骤”，不再提供直接创建并登录路径。
- 移动端保持单列字段、可滚动弹窗和至少 38px 的主要点击区域。

## 3. 社交入口

- 顶栏统一使用“社交”入口；待处理好友申请以数字徽标显示。
- 好友码可分享，但不暴露邮箱。
- 版图频道入口位于世界规则与国家列表之间，不需要滚动到页面底部寻找。
- 聊天默认收起；新消息显示未读徽标，打开后标记为已读。
- 聊天区区分本人/他人气泡，显示当前执政国提示、字数、发送状态和键盘快捷键。

## 4. 权限与发布准入

- 好友和聊天写入继续经过 Social Gateway；浏览器不直接写权威表。
- 世界消息附带的执政国必须由 Social Gateway 根据有效控制租约核验，客户端不能冒用其他国家身份。
- 频道读取必须先验证 `world_memberships`。
- 新注册流程发布前至少验证：表单校验、OTP 发送态、错误态、已验证密码登录、移动端无横向溢出。
- 真实邮箱验证码由外部邮件投递，自动化测试只验证 Appwrite 请求和 UI 状态；不可在代码或 CI 中硬编码验证码。
- 完整认证与响应式社交界面将生产 CSS 维持在约 89 KiB；体积门禁上限为 92 KiB，后续新增样式需先复用或清理现有规则。

## 5. 阶段状态

- `RELEASED`：账号/社交视觉、点击反馈、强制验证注册、好友申请徽标、精致版图频道入口。
- `NEXT`：聊天屏蔽/举报、消息保留策略、在线状态与好友私聊产品评审。
- `DEPENDENCY`：共享世界 Phase 5 的 Resolver 完成前，聊天和好友不得直接改变世界状态。
