# 后续完善路线图

状态含义：`DONE` 已完成，`PARTIAL` 已完成一部分，`NEXT` 下一阶段，`PLANNED` 已规划，`BACKLOG` 需求池。

## Phase 0：正确性底座 — DONE

- 动态玩家国取代回合内硬编码玩家。
- 中立所有权成为正式合约。
- 事件选择改为不可变、单次结算。
- AI 面对中立边境不再崩溃。
- 四层运行时 store 补丁收敛为显式回合管线。
- GameState 正式包含战略焦点、AI 记忆和国运元数据。
- 建立状态不变量审计和完整发布测试门禁。

## Phase 1：状态事务化 — DONE

目标：玩家操作不再依赖“原地修改对象 + 浅拷贝顶层”。

- `DONE` 建设、征兵、外交、政策、法律、省份开发拆成独立 action service。
- `DONE` 每个 action 返回 `{ state, ok, messages }`，不访问 Zustand；失败返回原状态引用。
- `DONE` 资源失败、行动点、所有权、敌国建筑、负数征兵和战争归属均有回归测试。
- `DONE` `gameStore.ts` 收敛为薄适配层；剧本目录、存档和五类 action 分离。
- `DONE` 存档压缩改为复制后规范化，不再污染运行中状态。

验收：所有窄 Zustand selector 都能依靠正确引用变化刷新；action service 可在 Node 测试中独立运行。

## Phase 2：回合引擎纯化 — IN PROGRESS

- `DONE` 正式 `processTurn` 切换到输入隔离实现，`processTurnPure` 成为兼容别名。
- `DONE` 删除仅供结果对照的 `processTurnLegacy` 重复实现；迁移期差分测试改为正式入口契约测试。
- `DONE` AI 年结拆入 `engine/aiTurnPhase.ts`，世界动态、领土变化与叛军衰减拆入 `engine/turnIntel.ts`。
- `DONE` 失败与叛乱后果拆入 `engine/turnConsequences.ts`；成功条件只由 `gameplay/ambitions.ts` 判定，不再并存 7 省/2000 金旧阈值。
- `DONE` 领土变化改在叛乱判定与叛军衰减后采集，新叛乱和五年后归顺都进入当年年报。
- `DONE` 删除 `logicGuard` 的后置规则补丁；正式 `processTurn` 直接负责输入隔离和正确报告，兼容名只做无逻辑别名。
- `DONE` 王朝、每回合法律与战争阶段改为“纯计算结果 + 单次合并”；战争纯结果由完整战斗差分测试锁定。
- `DONE` 经济、战争、事件效果与事件记录的旧可变 API 已收敛为纯实现的薄包装，不再保留第二份完整规则体。
- `DONE` 实体 ID 由 `Date.now()`/进程计数器迁移为 v5 存档内序列；相同初态、种子和动作历史产生完全相同的可序列化状态。
- `DONE` 建筑结算落实“基础产出 × 等级 × 地形修正”，补齐此前未入账的食物和木材产出。
- 按经济、人口、政治、外交、战争、事件、AI 顺序逐模块移除 mutation。
- 每迁移一个模块，补“输入不污染 + 确定性 + 状态不变量”测试。
- `DONE` 状态克隆优先使用 `structuredClone`，旧隔离包装不再重复克隆。
- `DONE` AI 长期战略改为单次索引，世界回合实测约 0.34–0.38 秒。
- `DONE` 增加经典、区域、世界自动模拟与回合性能预算。
- `DONE` 增加 5 样本性能基准；当前 P95 约为经典 11ms、区域 168ms、世界 266ms/回合（设备相关，仅作本机基线）。
- `DONE` 区域与完整世界加入全状态确定性重放，包含相同玩家事件选择与逐回合不变量检查。

验收：完整回合不修改输入；相同种子与动作序列产生相同哈希；50 回合世界模拟性能不退化。当前输入隔离、完整状态确定性和三规模性能预算已锁定，世界长程玩法平衡仍待扩展。
