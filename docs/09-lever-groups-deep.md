# Imperium Aeternum — 杆组细讲（Lever Group Deep Dive）

> 配套文件：`docs/09-lever-groups.svg`（可视化总图）
> 设计圣经原话：**"国家扩张越大，治理越困难。短期强权可以快速成功，但长期可能崩溃。"**
> 杆组就是把这句话变成数学的载体。

---

## 一、杆组到底是什么

普通策略游戏的按钮是**开关**：点了就发生，不点就不发生，按钮之间互不影响。
Imperium 的按钮是**杆**：拉一根，另外几根必然动。杆之间用齿轮咬合，你拉税率的杆，民众满意度的杆就被往下压。

**杆组 = 一组用齿轮咬合的联动杠杆 + 一根回弹弹簧（阻尼）。**

玩家要学的不是"按哪个按钮赢"，而是"什么时候拉哪根杆、拉多狠、什么时候松手"。这是节奏游戏，不是优化游戏。

---

## 二、为什么用"杆"不用"树"或"图"

杆有三条性质是树/图没有的：

| 性质 | 杆的表现 | 游戏含义 |
|------|---------|---------|
| **行程** | 杆只能拉到某个刻度，拉过头会卡死 | 税率顶 50% → 民众暴动 |
| **回弹** | 松手后杆会自己回弹，但回不到原位（有磨损） | 厌战降下来要时间，且留下伤痕 |
| **联动** | 拉甲杆，乙杆丙杆按齿轮比跟着动 | 征兵→粮耗→人口→税基全连锁 |

"树"只能表达层级，"图"只能表达连接，**杆**能表达**力度、行程、回弹**——这才是策略游戏数值该有的形态。

## 三、五大杆组逐一细讲

### 1. 财政杆组（economy.ts）

```
玩家可拉：税率（0-50%）、建设预算、贸易路线
    │
    ├─►【短期收益】国库金 +
    │
    ├─►【联动 1】民众满意度 − ─► 不满 + ─► 叛乱风险 +
    │
    ├─►【联动 2】商人满意度 − ─► 贸易收入 −（反噬短期）
    │
    ├─►【联动 3】人口增长 −（跨回合）─► 下一回合税基 −
    │
    └─►【阻尼】腐败吞收入（省越多吞得越多）
```

**阻尼公式**：`corruptionLoss = taxIncome × (corruption/100)`，腐败随省数上升。
**玩家感受**：一开始加税很爽，10 回合后金越收越少，才发现腐败吃掉了一半。

### 2. 军事杆组（military.ts）

```
玩家可拉：征兵、宣战、动员
    │
    ├─►【短期收益】兵力 +、省 +
    │
    ├─►【联动 1】粮耗 + ─► 粮仓见底 ─► 饥荒
    │
    ├─►【联动 2】厌战 + ─► 稳定 − ─► 叛乱 +
    │
    ├─►【联动 3】人口 −（征兵抽走农民）─► 税基 −
    │
    └─►【阻尼】厌战达到 70+ 触发厌战事件链，80+ 触发兵变
```

**阻尼公式**：`warExhaustion += warCostPerTurn(armySize, warCount)`，每回合累积。
**玩家感受**：连打三场胜仗，第四回合国内叛乱四起——因为厌战和粮耗滞后发作。

### 3. 扩张杆组（culture.ts + politics.ts）

```
玩家可拉：占省、附庸、建设
    │
    ├─►【短期收益】省 +、资源 +
    │
    ├─►【联动 1】行政超载 ─► overExtensionPenalty
    │     └─► 税收效率 −、稳定 −
    │
    ├─►【联动 2】文化冲突 ─► 同化度低 ─► 叛乱风险 +
    │
    ├─►【联动 3】合法性 −（新征服省不稳）
    │
    └─►【阻尼】maxManageableProvinces = admin等级 × 系数
              超过该数每省扣稳定/行政
```

**阻尼公式**：`overExtensionPenalty = max(0, provinceCount − maxManageable) × 2`
**玩家感受**：打下第 8 省时突然行政瘫痪，第 9 省直接叛乱——因为行政等级只够管 7 省。

### 4. 改革杆组（politics.ts）

```
玩家可拉：政策、法律、政体切换
    │
    ├─►【短期收益】解锁加成（贸易+、行政+等）
    │
    ├─►【联动 1】派系反弹（受损派系满意度 −）
    │
    ├─►【联动 2】改革冲击（3 回合内稳定 −）
    │
    ├─►【联动 3】合法性波动
    │
    └─►【阻尼】reformCostMultiplier（政体决定改革成本）
              君主制 1.3×、共和国 0.9×、僭主制 0.8×
```

**玩家感受**：一次性推 3 个改革，派系全崩；分 5 回合慢慢推，反而稳。

### 5. 外交杆组（diplomacy.ts）

```
玩家可拉：结盟、制裁、宣战、间谍
    │
    ├─►【短期收益】盟友 +、贸易依赖 +
    │
    ├─►【联动 1】被包围网（威胁值高的国家联合抵制）
    │
    ├─►【联动 2】信任值 −（反复违约）
    │
    ├─►【联动 3】贸易依赖风险（断交则经济崩）
    │
    └─►【阻尼】威胁值超阈值触发联军反制
```

**玩家感受**：吃掉第 3 个小国后，5 个大国突然结盟打你——因为你威胁值爆表。

## 四、三种耦合形态（配点图理解）

### 形态 A — 串型杆组（链式反噬）

**特征**：单向链条，末端必然炸。玩家要学会**中途松手**。

```
   ┌─────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
   │ 宣战  │───►│ 厌战 +  │───►│ 稳定 -   │───►│ 叛乱 +  │
   └─────────┘    └──────────┘    └──────────┘    └──────────┘
                        │
                        └────────► 粮耗 + ──► 人口 -
```

**代码体现**：`declareWar` → `settleWars` 累积 warExhaustion → `settlePolitics` 扣稳定 → `settleCultureReligion` 升叛乱风险。

### 形态 B — 环型杆组（自反馈回路）

**特征**：杆有"记忆"。今天的爽=明天的痛。这是 Imperium 区别于涂色游戏的核心。

```
        税率 ↑
       ╱        ╲
      ▼          ▼
   金 +        民众满意度 −
      │          │
      │          ▼
      │        人口增长 −
      │          │
      └◄─────────┘
        税基 ↓  ← 下一回合金 -（反噬）
```

**代码体现**：`settleEconomy` 算税 → `settlePopulation` 算人口增长（读满意度）→ 下一回合 `settleEconomy` 税基变小。环跨回合闭合。

### 形态 C — 对冲杆组（双杆博弈）

**特征**：没有免费午餐。两根杆方向相反，玩家选**节奏**而非选"对错"。

```
   ┌── 拉杆甲：安抚民众 ──► 满意度 + │ 成本：金 - │ 副作用：腐败 +
   │
   └── 拉杆乙：镇压叛乱 ──► 叛乱 -   │ 成本：粮 - │ 副作用：军方 + / 民众 -

        甲、乙不能同时拉满 → 玩家做权衡
```

**代码体现**：`appease` 扣金加满意度但加腐败；`suppress` 扣粮降叛乱但加军方满意度降民众。两行动在 `gameStore` 互斥预算。

## 五、杆组在代码里怎么找

每个杆组 = **一个 engine 文件 + 一组 formulas 函数**。联动通过 `settleXxx` 互调实现：

```
turn.ts  processTurn(state)
   ├── settleEconomy          ← 财政杆组（拉：税率/建设）
   ├── settlePopulation       ← 人口杆组（读经济结果）
   ├── settlePolitics         ← 政治杆组（读不满度）
   ├── settleCultureReligion  ← 文化杆组（读稳定度）
   ├── settleWars             ← 军事杆组（读厌战/粮）
   └── buildReport            ← 把所有杆的位移汇总给玩家看
```

**阻尼系数随省数上升**，正是"扩张越大治理越难"的代码化：

| 杆组 | 阻尼函数 | 文件 |
|------|---------|------|
| 财政 | `corruptionLoss = tax × corruption/100` | economy.ts |
| 军事 | `warExhaustion += warCostPerTurn(...)` | military.ts |
| 扩张 | `overExtensionPenalty = max(0, provCount − maxManageable) × 2` | politics.ts |
| 改革 | `reformCostMultiplier`（政体定） | governments.ts |
| 外交 | 威胁值超阈值触发联军 | diplomacy.ts |

杆不会断，但会越来越重。

## 六、玩家心智模型（最终目标）

玩家不需要看公式。玩家感受到的是：

1. 拉某根杆很爽（金多了、省多了、军多了）
2. 但下一回合杆变重了（腐败涨了、不满涨了、厌战涨了）
3. 必须松一根、拉另一根，做**节奏切换**
4. 长期把任何一根顶死 = 崩盘

这就是 Imperium 的核心乐趣：**不是赢，是撑住。**

永恒帝国胜利条件（连续 200 回合稳定）就是杆组设计的终极考验——你要同时压住五根杆不让任何一根爆，持续 200 回合。

## 七、本轮杆组闭环进度

| 杆组 | 主杆 | 联动 | 阻尼 | 状态 |
|------|------|------|------|------|
| 财政 | 税率/建设/贸易 | ✅ 满意度/腐败/贸易 | ✅ 腐败吞收入 | ✅ 闭环 |
| 军事 | 征兵/宣战 | ✅ 厌战/粮耗/人口 | ✅ 厌战→叛乱 + 和约条件 | ✅ 闭环 |
| 扩张 | 占省/附庸 | ✅ 行政超载/文化冲突/合法性 | ✅ overExtension + 分裂分级 | ✅ 闭环 |
| 改革 | 政策/法律 | ✅ 派系反弹/改革冲击 | ✅ reformCost + 政体被动 | ✅ 闭环 |
| 外交 | 结盟/制裁 | ✅ 信任/威胁值/贸易依赖 | ✅ 联军反制（威胁≥70 且 ≥3 邻国抵制） | ✅ 闭环 |

外交杆组的"联军反制"阻尼待补——这是下一轮螺旋的目标。
