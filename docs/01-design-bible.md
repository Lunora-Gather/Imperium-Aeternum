## 1. 游戏一句话概念

> **治理一个国家数百年：扩张越快，崩溃越早；真正的胜利是让帝国永续运转。**

（30 字以内，已满足 §3.2 验证点①）

## 2. 玩家扮演什么

玩家是一个**国家统治者**（不是将军，也不是单纯的内政官）。

视角：坐在首都的王座/议事厅/议政殿里，面对一面面信息面板与一张省份战略地图，做国家层面的决策。

具体身份随政体而变：
- 君主制 → 国王/女王
- 共和国 → 首席执政官
- 神权制 → 大祭司/教宗
- 军政府 → 大将军
- 帝国制 → 皇帝

玩家不直接操控每一个士兵或每一个农民，而是通过**政策、预算、建设、外交、战争**等高层决策影响国家。

玩家不是无成本的"上帝"：每个决策消耗**行动点（每回合 3 点）**，且会触发各阶层、各派系、邻国的反馈。

## 3. 核心乐趣

 Imperium Aeternum 的乐趣不是"涂色征服"，而是以下五种张力带来的策略愉悦：

1. **短期收益 vs 长期风险的权衡**
   - 高税快富，但民心崩；快速扩张，但治理崩。
   - 玩家每回合都在做"现在爽"还是"百年后还在"的选择。

2. **多方制衡的政治博弈**
   - 贵族、商人、军方、民众、神职互相牵制。
   - 安抚一方常得罪另一方，没有"全赢"政策。

3. **扩张的悖论**
   - 占的越多，文化/宗教/行政压力越大，叛乱越多。
   - 玩家会主动问自己："这块地我真的要吗？"

4. **危机驱动的叙事**
   - 粮荒、瘟疫、王位继承、贵族逼宫、边境冲突……
   - 每个事件都是一次"两难选择"，构成玩家的故事记忆。

5. **塑造国家性格的成就感**
   - 玩家的长期行为会把国家塑成"军国主义/商业共和国/福利国家…"
   - 不是选预设 buff，而是"我做了什么，于是我的国家成了什么"。

**反面声明（不追求的乐趣）**：
- ❌ 不追求战术微操的战斗快感
- ❌ 不追求视觉奇观与美术震撼
- ❌ 不追求"全图涂色"的征服爽感
- ❌ 不追求无脑滚雪球的资源堆叠

## 4. 核心玩法循环

每回合（= 1 年）玩家走以下 9 步循环：

```
┌─────────────────────────────────────────────────────────────┐
│  1. 查看国家状态                                            │
│     ↓                                                       │
│  2. 分析财政/人口/粮食/稳定/外交/军事                       │
│     ↓                                                       │
│  3. 制定政策（消耗行动点）                                  │
│     ↓                                                       │
│  4. 分配预算（建设/科研/外交/军事）                        │
│     ↓                                                       │
│  5. 建设省份（农田/道路/市场/兵营等）                       │
│     ↓                                                       │
│  6. 处理事件（弹窗，二选一/多选一）                         │
│     ↓                                                       │
│  7. 外交或战争（改善关系/结盟/宣战/停战）                   │
│     ↓                                                       │
│  8. 进入下一回合 → processTurn(gameState)                   │
│     ↓                                                       │
│  9. 结算国家变化 + 产生新问题/机会 → 回到 1                 │
└─────────────────────────────────────────────────────────────┘
```

**节奏说明**：
- 玩家在 1-7 步中可自由操作（受行动点限制）。
- 第 8 步点击"结束回合"后，引擎统一结算所有系统。
- 第 9 步生成 `TurnReport`，玩家下回合开局看到变化与警告。

## 5. MVP 范围

### 5.1 MVP 做什么

| 维度 | MVP 包含 |
|------|----------|
| 平台 | PC 浏览器（React + TS + Vite） |
| 视角 | 2D 面板 + 简单 SVG 地图 |
| 回合 | 固定 1 回合 = 1 年 |
| 省份 | 10 个固定省份 |
| 国家 | 5 个（玩家 1 + AI 4） |
| 政体 | 5 种（君主/共和/神权/军政府/帝国） |
| 人口阶层 | 7 类（农/工/商/兵/学/贵/神） |
| 资源 | 金、粮、木、铁、人口、劳动力、行政点、科研点、外交影响力、军事补给 |
| 行动点 | 每回合 3 点 |
| 行动类型 | 征/降税、4 种建设、征兵、研发、安抚、镇压、外交改善、宣战、改革 ≈ 14 种 |
| 科技 | 3 条 × 5 级 = 15 项 |
| 建筑 | 8-10 种 |
| 事件 | 25-30 个 |
| 政策 | 8-12 项 |
| 派系 | 5 个（贵族/商人/军方/民众/神职） |
| 国家性格 | 10 种倾向 |
| AI 国家 | 4 个，权重决策 |
| 战争 | 战略层结算（无即时战斗） |
| 外交 | 关系值 + 条约 + 5 种行动 |
| 胜利 | 4 种 |
| 失败 | 6 种 |
| 存档 | localStorage 单槽，含 version |

### 5.2 MVP 不做什么（红线，见 `00-project-plan.md §11`）

- ❌ 即时战斗 / 战斗演出
- ❌ 大地图 / 随机地图
- ❌ 多语言（先中文）
- ❌ 多存档槽
- ❌ 联机
- ❌ 美术资源（纯 CSS + SVG 几何图形）
- ❌ 音效音乐
- ❌ 复杂外交链
- ❌ 科技超 3×5
- ❌ 事件超 30
- ❌ 省份超 12
- ❌ AI 国家超 4
- ❌ 回合长度切换
- ❌ 自定义国家

## 6. 完整版愿景

MVP 验证玩法后，完整版方向（不在本 MVP 实现）：

- **多剧本**：青铜时代崛起、古典帝国、中世纪王国、近代革命等不同起点。
- **随机地图 + 地形生成**：每局不同地理。
- **战斗演出层**：关键战役有简易动画与将领技能。
- **复杂外交链**：联姻、附庸树、同盟体系、间谍网。
- **更细科技树**：每条 10-15 级，含分支与互斥。
- **海洋与海军**：沿海省份、海战、贸易航线。
- **宗教改革 / 文化复兴**：思潮系统。
- **王朝系统**：统治者 aging、继承人、摄政。
- **多存档 + 云存档**。
- **Mod 支持**：数据全 JSON 化，玩家可加事件/国家。
- **移植 Godot/Unity**：玩法成熟后转引擎做美术升级。

## 7. 核心系统总览

| # | 系统 | 一句话目标 | MVP 简化度 |
|---|------|------------|------------|
| 1 | 政治 | 让政体/合法性/稳定/派系形成制衡 | 中 |
| 2 | 经济 | 金粮物资 + 税收/腐败/贸易的收支循环 | 中 |
| 3 | 人口 | 7 阶层各有满意度与政治影响力 | 中 |
| 4 | 军事 | 战略层结算，战争昂贵且有代价 | 高（无战斗演出） |
| 5 | 外交 | 关系/条约/威胁/信任构成的多维关系 | 中 |
| 6 | 科技 | 3×5 科技树，提供长期增益 | 高（小树） |
| 7 | 文化宗教 | 同化/忠诚/叛乱压力 | 高（简化） |
| 8 | 事件 | 数据驱动，25-30 事件构成叙事 | 中 |
| 9 | 国家性格 | 玩家行为塑造国家倾向，达阈值获特性 | 中 |
| 10 | AI 国家 | 权重决策，4 个 AI 各有性格 | 中 |

每个系统的详细规则、公式、事件触发见 `docs/02-system-rules.md` 与 `docs/formulas.md`（阶段 2 产出）。

## 8. MVP 中必须砍掉或简化的内容（逐系统）

| 系统 | 砍/简化项 | 理由 |
|------|-----------|------|
| 政治 | 王朝继承、摄政、宪法起草 | MVP 用合法性数字代替 |
| 经济 | 通货膨胀、汇率、多币种 | 单币种（金）足够 |
| 经济 | 贸易路线可视化 | 用贸易依赖度数字代替 |
| 人口 | 阶层内细分（如工人→矿工/匠人） | 7 大类足够 |
| 人口 | 人口迁徙地图 | 用增长率数字代替 |
| 军事 | 即时战斗、战术指令、单位移动 | 战略层一次结算 |
| 军事 | 海军、攻城器械、多兵种克制 | 单兵种"军队"抽象 |
| 外交 | 联姻、间谍网、附庸树 | 仅关系/条约/5 行动 |
| 科技 | 科技分支互斥、前置网 | 3 条线性 × 5 级 |
| 文化宗教 | 多宗教多文化并存 | 每省主流 1 文化 + 1 宗教 |
| 事件 | 事件链（A→B→C 跨回合） | MVP 事件单回合结算 |
| 国家性格 | 性格随时代演变 | 一次塑造，可缓慢漂移 |
| AI | 多层目标规划、学习型 AI | 静态权重 + 性格修正 |
| 地图 | 地形渲染、海拔、河流 | 每省 1 terrain 标签 |
| UI | 动画、过渡、主题切换 | 静态面板 + SVG |
| 存档 | 多槽、云存档、回滚 | 单槽 + version migration |

## 9. 下一阶段要产出的数据表预告

阶段 3（拆 3a/3b）将产出以下数据表，对应 `src/data/*.ts`：

**3a 核心数据表**：
- `governments.ts` — 5 政体定义（buff/debuff）
- `nations.ts` — 5 国初始定义（属性/资源/政体/性格）
- `provinces.ts` — 10 省份定义（地形/资源/人口/归属）
- `buildings.ts` — 8-10 建筑（成本/产出/前置）
- `technologies.ts` — 3×5 科技（效果/前置）
- `policies.ts` — 8-12 政策（效果/阶层反应）

**3b 内容数据表**：
- `factions.ts` — 5 派系（关注点/影响力/初始满意度）
- `national-characters.ts` — 10 国家性格倾向（阈值/加成/副作用）
- `events.ts` — 25-30 事件（触发条件/选项/效果/AI 权重）
- AI 国家决策权重表（并入 nations 或单独 `ai-weights.ts`）

阶段 2（系统规则）将先定每个数据表的字段语义与取值范围，阶段 3 据此填数。

**补：胜利与失败条件**（MVP 锁定）

4 胜利：
1. **征服胜利**：控制 ≥ 7/10 省份且稳定度 ≥ 40
2. **经济胜利**：国库 ≥ 2000 且年贸易收入 ≥ 500，连续 10 回合
3. **文化胜利**：外交影响力 ≥ 150 且 ≥ 3 国与玩家关系 ≥ 70
4. **永恒帝国胜利**：连续 200 回合（200 年）未触发任何失败条件

6 失败：
1. **国库破产**：连续 3 回合国库 ≤ 0
2. **全国崩溃**：全国平均稳定度 ≤ 10 连续 3 回合
3. **首都被占**：玩家首都省份 ownerId ≠ 玩家
4. **革命成功**：任一省份叛乱值 ≥ 100 且驻军 = 0
5. **国家分裂**：≥ 3 省份同时叛乱且独立
6. **合法性归零**：合法性 ≤ 0
