## 第一层：核心玩法缺口（不做则游戏不好玩）

### P1-1 🔥 省份叛乱实际执行逻辑 [M]
**现状**：`judgeVictory` 检测 `rebellionRisk >= 100 && garrison === 0` 的省份设 `ownerId = rebel_xxx`，但：
- 叛乱省份变为 `rebel_xxx` 后无对应 Nation 对象，后续引擎遍历 `nations[id]` 会 undefined
- 叛乱省份的 adjacent 省份不连锁反应（叛乱应可蔓延）
- 叛乱省份的军队/建筑/人口不变（应损失兵力/破坏建筑/人口逃散）

**方案**：
1. 叛乱省份创建临时 Nation（`id: rebel_p01, name: 叛军·xx, tier: D, ...`），加入 `state.nations`
2. 叛乱省份驻军清零、随机破坏 1-2 建筑、人口 ×0.7
3. 相邻同文化省份若 `rebellionRisk > 60`，有 30% 概率连锁叛乱
4. 叛军 Nation 每回合衰减，5 回合后若未被镇压则自动归顺（稳定度恢复）
5. Chronicle 记录"叛乱爆发"里程碑

### P1-2 🔥 国家分裂机制 [M]
**现状**：`judgeVictory` 仅设 `fail_split`，无实际分裂逻辑——3+ 省叛乱直接游戏结束，玩家无法挽回。

**方案**：
1. 3+ 省叛乱不直接 fail，而是触发"内战"状态——叛乱省份组成叛军国家
2. 玩家可选择镇压（战争）或谈判（让步+合法性-10+割1省）
3. 内战期间稳定度每回合 -5、税收减半
4. 若叛军占领首都才 fail_split
5. 镇压成功后 Chronicle 记录"平定叛乱"

### P1-3 🔥 政体切换实际效果 [M]
**现状**：`changeGovernment()` 仅改 `government.type`，无实际效果差异——君主制/共和国/军政府玩起来一样。

**方案**：每种政体有持续被动效果（每回合结算）：
| 政体 | 每回合效果 | 限制 |
|------|-----------|------|
| 君主制 | 合法性 +0.2/回合，贵族满意度 +0.5 | 改革成本 ×1.5，商人满意度 -0.3 |
| 共和国 | 贸易收入 +10%，科技 +0.3/回合 | 战争动员 -20%，派系斗争（随机 -1 满意度）|
| 神权制 | 神职满意度 +1，同宗教省份忠诚 +0.5 | 异教省份不满 +1，科技 -0.2/回合 |
| 军政府 | 征兵成本 -30%，军队士气 +5 | 民众满意度 -1/回合，外交关系 -10 |
| 帝国制 | 行政能力 +10，可管省 +2 | 腐败 +0.3/回合，派系影响力斗争 |

在 `settlePolitics` 中按 `government.type` 应用被动效果。

### P1-4 🔥 国家性格实际加成生效 [S]
**现状**：`national-characters.ts` 定义了 11 种性格的加成/副作用，但 `settleEconomy`/`settlePolitics`/`settlePopulation` 未读取 `nation.activeCharacterBonuses`——性格仅 UI 显示，引擎不生效。

**方案**：在对应引擎中读取 `nation.activeCharacterBonuses`：
- `settleEconomy`：`militarist` → 征兵成本 -20%；`commerce` → 贸易收入 +25%
- `settlePolitics`：`welfare` → 民众满意度 +0.5/回合；`despotic` → 镇压效率 +30%
- `settlePopulation`：`expansionist` → 新省份同化 +1/回合
- `military.ts`：`militarist` → 军队士气 +10%

### P1-5 🔥 贸易协定实际收益 [S]
**现状**：`settleDiplomacy` 可设 `treaty = 'trade'`，但 `settleEconomy` 的 `tradeDeals` 仅按条约数量算收入——贸易协定无差异化，和谁贸易都一样。

**方案**：
1. 贸易收入 = 基础 × (1 + 对方GDP/1000) × 关系修正 × 距离修正
2. 与大国贸易收入高但依赖风险大（对方关系变负时收入损失大）
3. 与小国贸易收入低但稳定
4. UI 显示每个贸易伙伴的预估收入

### P1-6 🔥 战争和约条件 [M]
**现状**：`makePeace` 仅停战 10 年 + 关系 +10，无割地/赔款——战争无胜负感。

**方案**：
1. 进攻方占优（progress > 60）→ 防守方割 1 省 + 赔款 200 金
2. 防守方占优（progress < 30）→ 进攻方赔款 100 金 + 厌战 +20
3. 均势（30-60）→ 白平和，双方各 -5 合法性
4. 玩家可选择接受/拒绝（拒绝则继续战争但厌战 +10）
5. AI 根据战况自动接受/拒绝

## 第二层：体验闭环缺口（不做则玩家困惑）

### P2-1 ⚡ 事件选项效果预览 [S]
**现状**：EventModal 选项仅显示效果 Tag 摘要，但 `factionSat` 效果不翻译（显示原始 faction id）。

**方案**：`effectSummary()` 中 `factionSat` 翻译为中文（贵族+8 → "贵族满意+8"），与 PoliticsScreen 派系反应预览一致。

### P2-2 ⚡ 省份归属变化通知 [S]
**现状**：战争占领省份后无明确通知——玩家需手动切到省份页才发现多了/少了省。

**方案**：回合报告加 `provinceChanges: { id: string; from: string; to: string }[]`，TurnReportScreen 显示"获得/失去省份"叙事。

### P2-3 ⚡ AI 国家行为可见性 [M]
**现状**：AI 每回合做什么玩家完全看不到——不知道邻国在扩军/结盟/宣战。

**方案**：
1. TurnReport 加 `worldEvents: string[]`——重要 AI 行为摘要
2. AI 宣战/结盟/灭国记入 worldEvents（仅玩家邻国或相关事件）
3. TurnReportScreen 加"天下大势"区显示 worldEvents

### P2-4 ⚡ 科技前置依赖可视化 [S]
**现状**：TechnologyScreen 科技卡片无前置线连接——玩家不知道先学什么才能解锁。

**方案**：E14 已有分支连接线，但前置科技依赖（如 admin_lv3 需要 admin_lv2）未可视化。加虚线箭头从前置指向后续。

### P2-5 ⚡ 外交条约到期提醒 [S]
**现状**：停战条约 `truceTurns` 每回合递减，但到期时无提醒——玩家不知道何时可以再宣战。

**方案**：
1. DiplomacyScreen 停战国家显示剩余回合数
2. 停战到期当回合自动通知（LogToast）
3. MilitaryScreen 宣战区停战国家显示"X 回合后可宣"

### P2-6 ⚡ 省份详情跳转 [S]
**现状**：WorldMap/Dashboard 点击省份无法跳转到 ProvinceScreen 详情。

**方案**：WorldMap 省份圆点 onClick → setTab('province') + setSelectProvince(id)。Dashboard 危机省同样可点击跳转。

### P2-7 ⚡ 建筑拆除功能 [S]
**现状**：建筑只能建/升级不能拆——错误建设无法挽回。

**方案**：ProvinceScreen 建筑卡片加"拆除"按钮（返还 30% 金，清除建筑实例）。

### P2-8 ⚡ 多存档槽位 [M]
**现状**：仅 1 个 localStorage 存档位——无法保留关键节点。

**方案**：
1. 存档改为 5 槽位（`imperium-save-0` ~ `imperium-save-4`）
2. SaveLoadScreen 显示 5 槽位卡片（回合/国名/日期/截图摘要）
3. 自动存档每 10 回合到槽位 0

## 第四层：架构健壮缺口（不做则后期难维护）

### P4-1 🏗️ store 浅拷贝全渲染优化 [L]
**现状**：`set({ state: { ...st.state } })` 浅拷贝触发所有订阅组件重渲染——12 个 tab 页每次操作都重渲染。

**方案**：
1. 用 Zustand selector 精确订阅（`useGameStore(s => s.state.nations[pid])`）
2. 或引入 Immer 不可变更新（`produce(state, draft => { ... })`）
3. 关键屏幕用 `React.memo` 包裹
4. 预期：操作响应从 ~50ms 降至 ~5ms

### P4-2 🏗️ 存档版本迁移 [M]
**现状**：`migrations` 数组为空——旧存档加载后缺新字段（embargoedRoutes/tech.culture/battleReports/chronicle/warProgress/factionDelta）直接 undefined。

**方案**：
1. 实现 v1→v2 迁移：补 `embargoedRoutes: []`、`tech.culture: 0`、`chronicle: []`、`battleReports: []`
2. 实现 v2→v3 迁移：补 `TurnReport.warProgress: []`、`TurnReport.factionDelta: []`
3. 每次 TurnReport 加字段都加迁移
4. SAVE_VERSION 递增

### P4-3 🏗️ 引擎纯函数化 [M]
**现状**：引擎函数直接 mutate Nation/Province 对象——无法做回滚/重放/调试。

**方案**：
1. 核心引擎改为 `(state) => newState` 纯函数（或 Immer produce）
2. processTurn 返回新 state 而非 mutate
3. 好处：可做撤销操作、可做时间线回放、可做确定性重放测试
4. 渐进式：先改 turn.ts，再改各子引擎

### P4-4 🧪 引擎单元测试扩充 [L]
**现状**：50 测试以烟雾测试为主，无针对性引擎逻辑测试。

**方案**：每个引擎至少 5 个针对性测试：
- `economy.test.ts`：高税降民心、腐败扣收入、禁运路线不产收益、贸易依赖扣成
- `population.test.ts`：粮食不足降增长、战争降增长、安抚提满意度
- `politics.test.ts`：政体被动效果、改革派系反应、法律每回合效果、超管惩罚
- `military.test.ts`：前线军队才进攻、调动合并加权、求和停战、战报生成
- `diplomacy.test.ts`：关系漂移、间谍成功率、联姻效果、文化输出效果
- `events.test.ts`：触发条件、冷却、唯一事件、链式触发、AI 选项权重

### P4-5 🏗️ TypeScript 严格类型 [S]
**现状**：部分引擎函数参数用 `any` 或缺返回类型。

**方案**：
1. 所有引擎函数补返回类型
2. 消除 `as` 类型断言（改用类型守卫）
3. `NationTier` / `TechBranch` / `TreatyType` 等用 union type 不用 string
4. `npm run typecheck --strict` 通过

### P4-6 🏗️ 错误边界 [S]
**现状**：React 组件无 ErrorBoundary——引擎异常直接白屏。

**方案**：
1. App.tsx 包裹 ErrorBoundary
2. 错误时显示"国运不济"提示 + 重新开始按钮
3. 错误信息上报 console（开发模式）

## 核心设计原则（贯穿所有补充）

1. **扩张有代价**：每加 1 省带来超管惩罚 + 文化冲突 + 行政负担，不是越多越好
2. **改革有反弹**：每次改革触发派系反应，短期阵痛换长期收益
3. **战争有成本**：军费/厌战/人口损失/稳定度下降，胜利也伤国
4. **稳定是永恒**：200 年太平才是真胜利，短期强权终将崩溃
5. **信息要透明**：玩家每个决策点都能看到预估效果和风险提示
6. **AI 要可信**：AI 行为可见、逻辑可理解、不作弊
